# remove-auto-extension-sync

## What

移除 V2 番源仓库的启动自动同步能力。后续应用启动、进入主界面、仓库远端数据加载完成时，都不再自动下载 onlyRemote 插件，也不再自动更新 hasUpdate 插件。

本次变更保留用户主动行为：

- 用户仍可在插件管理页手动刷新远端仓库列表。
- 用户仍可点击远端插件进行安装。
- 用户仍可点击有更新的插件进行手动更新。
- 用户仍可长按已安装的非系统 JS 插件并删除。

同时排查并修复插件管理中的边界条件，重点是删除成功但 UI/状态仍显示插件存在、删除失败后状态误判、下载/更新覆盖文件失败、索引与磁盘文件不一致等问题。

## Why

当前 `ExtensionRepoController.fireAutoSync()` 会在主界面创建后被 `Scheduler.runOnMainActivityCreate()` 调用，并在 `SettingPreferences.sourceAutoSync` 默认开启时自动同步远端仓库插件。这个行为会在用户未明确确认的情况下触发网络下载、插件更新和本地文件替换，容易带来不可预期的插件版本变化。

插件管理还存在一些一致性风险。例如删除流程依赖 `JsExtensionProviderV2.delete()` 同时删除文件与更新索引，如果删除结果、索引更新、状态流刷新之间出现异常或竞态，用户可能看到“删除成功但插件还在”的表现。下载/更新流程也需要明确文件复制、重命名、旧版本清理和失败回滚策略。

## Scope

包含：

- 删除启动自动同步调用路径。
- 删除或废弃自动同步偏好项在 UI 中的入口。
- 移除 `ExtensionRepoController` 中仅用于自动同步的状态与方法，或保证它们不再被调用。
- 梳理并加固 V2 JS 插件的安装、更新、删除状态一致性。
- 对插件管理页的删除/下载状态进行必要的 UI 状态修正。

不包含：

- 移除手动刷新仓库列表。
- 移除手动安装或手动更新插件。
- 改造 APK 安装插件的系统卸载流程。
- 变更仓库格式或远端 `index.jsonl` 协议。

## Acceptance Criteria

- 启动应用并进入主界面时，不会调用自动同步逻辑，也不会自动下载或更新仓库插件。
- 设置页不再展示“自动同步番源”开关，或该开关不再影响任何运行时行为。
- 用户手动刷新仓库列表仍可正常加载远端信息。
- 用户手动安装/更新远端 JS 插件仍可正常写入文件和索引，并触发列表刷新。
- 用户删除已安装 JS 插件后，插件文件、索引、内存缓存和插件列表状态保持一致。
- 删除目标不存在、索引残留、文件删除失败、索引更新失败、快速重复删除等场景有明确结果，不出现成功提示但列表仍显示为已安装。
- 下载/更新失败不会破坏已有可用插件；成功后不会留下同 key 的旧文件被索引继续引用。
