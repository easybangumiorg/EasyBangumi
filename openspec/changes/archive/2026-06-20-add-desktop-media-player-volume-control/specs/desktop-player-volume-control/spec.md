## ADDED Requirements

### Requirement: 桌面播放器提供可视化音量调节
系统 MUST 在桌面端播放器控制区提供音量调节控件，主控制栏常驻音量图标；当鼠标悬停在图标区域时 MUST 在图标上方展示临时音量窗口，并使用统一 `Seekbar` 在 0 到 100 的范围内设置播放音量与实时反馈当前值。

#### Scenario: 用户通过控件设置音量
- **WHEN** 用户悬停音量图标并在弹出的 `Seekbar` 中拖动到目标值
- **THEN** 系统 MUST 实时将预览值写入播放器并在 UI 中显示更新后的音量百分比

### Requirement: 桌面播放器支持静音切换
系统 MUST 提供静音开关，并在静音与非静音状态间可逆切换；从静音恢复时 MUST 恢复到静音前最近一次有效音量值。

#### Scenario: 用户点击静音按钮
- **WHEN** 用户点击静音按钮进入静音状态后再次点击
- **THEN** 系统 MUST 退出静音并恢复到进入静音前的音量值

### Requirement: 桌面播放器支持快捷音量操作
系统 MUST 支持快捷交互进行音量增减（包括快捷键或滚轮），并与控件调节共享同一状态与边界规则。

#### Scenario: 用户使用快捷交互增减音量
- **WHEN** 用户触发一次音量增加或减少快捷操作
- **THEN** 系统 MUST 按预设步进调整音量并将结果钳制在 0 到 100 范围内

### Requirement: 音量设置在本地持久化
系统 MUST 在桌面端本地保存最近一次音量值，并在播放器下次初始化时自动恢复该值；在拖动 `Seekbar` 高频更新场景下 MUST 采用节流持久化策略，结束拖动后立即提交最终值。

#### Scenario: 用户重启应用后继续播放
- **WHEN** 用户在一次会话中修改音量并重启应用后再次进入播放器
- **THEN** 系统 MUST 使用本地保存的音量值初始化播放器音量

### Requirement: UI 音量状态与底层播放状态保持一致
系统 MUST 保证 UI 中显示的音量与静音状态与底层播放器实际状态一致；当底层不支持音量写入时 MUST 提供可识别的降级表现。

#### Scenario: 底层音量写入失败
- **WHEN** 系统向底层播放器写入音量失败或返回不支持
- **THEN** 系统 MUST 保持 UI 与可用状态一致并阻止误导性成功展示
